# 拡張ロードマップ

会計事務所案件における本デモの拡張プラン。面談時に「今後どこまで広げられるか」を問われたときに参照する想定。

## Phase A：会計ソフト連携の拡充（実装1〜2週間）

### A-1. TKC FX クラウド CSV インポート
- TKC が提供する標準 CSV エクスポート形式に対応
- 既存の弥生 CSV パーサーと共通インタフェース化
- 仕訳から役員賞与の支給実績を抽出し、次期の届出書ドラフトに自動反映

### A-2. 弥生会計オンライン API 連携
- 弥生 API（OAuth 認証）で直接仕訳データを取得
- ユーザーが CSV エクスポートする手間を削減
- リアルタイム連携で前期実績ベースの提案が可能に

### A-3. freee / マネーフォワード連携
- 中小会計事務所の顧問先で増加中
- 各 API の認証フローを実装

## Phase B：帳票自動化の他項目展開（実装1か月）

事前確定届出書だけでなく、会計事務所が定型作業で扱う他の帳票へ展開。

- 外形標準課税関連書類（資本割、付加価値割）
- 法人税申告書別表（一部）
- 給与所得の源泉徴収票
- 法定調書合計表
- 各種税務署提出書類のフォーム入力 PDF 化

## Phase C：e-Tax XML 直接出力（実装2〜3週間）

- e-Tax の XML スキーマ（XSD）に準拠した XML 生成
- 利用者識別番号と電子証明書を介した直接提出（オプション）
- PDF 出力に加え「e-Tax 取り込み用 XML」のダウンロードを追加

## Phase D：議事録 OCR 入力（実装2週間）

- 紙やスキャン PDF の議事録から AI で構造化
- Claude のマルチモーダル対応（画像入力）を活用
- スマホで撮った議事録写真を直接アップロード

## Phase E：複数法人管理 UI（実装1か月）

会計事務所の顧問先 100 社規模での運用を想定。

- 顧問先マスタ管理
- 法人ごとの過去届出書履歴
- 提出期限のアラート機能
- スタッフ間でのレビュー・承認フロー
- Slack / Chatwork 通知連携

## Phase F：内部 RAG 検索（実装3週間）

会計事務所が蓄積する過去の届出書・議事録から類似事例を引いて提案する機能。

- Supabase pgvector + OpenAI Embeddings
- 「前期と同じ条件で作成」「過去類例から修正案を提示」
- 既存知識ベース（Shun が過去に検討した RAG 構成）の応用

## 各 Phase の優先度判断

| Phase | 開発工数 | 業務インパクト | 優先度 |
|---|---|---|---|
| A-1 (TKC CSV) | 小 | 大 | ★★★ |
| A-2 (弥生 API) | 中 | 大 | ★★★ |
| B (他帳票) | 大 | 大 | ★★ |
| C (e-Tax XML) | 中 | 中 | ★★ |
| D (OCR) | 小 | 中 | ★ |
| E (複数法人) | 大 | 大 | ★★ |
| F (RAG) | 中 | 中 | ★ |

優先度は、案件のスコープが「事前確定届出書を含む帳票自動化」に絞られている前提で、まず A-1, A-2 で会計ソフト連携を仕上げ、次に B で帳票の横展開、その後に E でスケールさせる順序が現実的。
