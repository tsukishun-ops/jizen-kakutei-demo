# 面談用デモスクリプト（5分版）

## 0. オープニング（30秒）
「お時間いただきありがとうございます。事前に案件文を拝見して、まず実際に動くプロトタイプを作ってきました。画面共有してもよろしいでしょうか」

→ Vercel URL を画面共有

## 1. 課題提示（45秒）
「会計事務所さんの実務を調べてみると、事前確定届出給与の届出書には 2 つの構造的な課題がありました。

ひとつは、国税庁が配布しているのが PDF のみで、Excel 版が存在しないこと。手入力か、PDF を変換するソフトが必要で、どちらも手間がかかります。

もうひとつは、議事録という非構造データから定型項目を抽出する作業が、毎期発生していること。

このふたつを Claude API と Python で一気通貫に解決するデモです」

## 2. 実演（2分30秒）

### Step 1
「実際にやってみます。役員マスタの CSV と、株主総会の決議メモを用意しました。決議メモはわざと自然な文章のまま、議事録から抜粋した形式にしています」

→ サンプル CSV をドラッグ&ドロップ  
→ 「サンプルを読み込む」ボタンで決議メモをテキストエリアに挿入  
→ 「AI で抽出」をクリック

### Step 2
「Claude Sonnet 4 が決議メモを読んで、届出書に必要な項目を構造化抽出しました。法人情報、決議日、職務執行期間、役員ごとの支給予定が、全て JSON Schema 通りに整理されています」

→ 抽出結果のフォームを見せる  
「ここで会計担当者さんが内容を確認して、必要なら修正できる UI を入れています。AI に全部任せきりにせず、最終確認のステップを必ず挟むのが業務システムとして大事なので」

→ 「修正して帳票生成」をクリック

### Step 3
「Excel と PDF が両方ダウンロードできます」

→ PDF を開いて、国税庁様式に正しく記入されているのを見せる  
→ Excel も開いて、編集可能な状態を見せる  
「PDF は国税庁の様式を直接埋めているので、そのまま税務署に提出可能。Excel は事務所内での加筆・確認用です」

## 3. 技術的な工夫（45秒）

「技術的に工夫した点を 3 つだけ。

ひとつは、Claude API への抽出指示で JSON Schema を厳密にバリデーションしていて、出力が必ず Pydantic でパース可能になっています。失敗時はリトライプロンプトで自己修正させる仕組みです。

ふたつめは、PDF が AcroForm 形式かどうかで実装を切り分けていて、フォーム入力 PDF なら pypdf でフィールド名指定、印刷専用 PDF なら reportlab で座標オーバーレイ、というハイブリッド構造になっています。

みっつめは、API コスト上限を 1 リクエストあたり 100 円にハードコードしていて、暴走を防いでいます」

## 4. 拡張プランと提案（45秒）

「現状はデモなので CSV 取り込みですが、本案件で想定されている TKC や弥生との連携については、TKC FX クラウドの CSV エクスポート対応がまず最短ルート、その次に弥生会計の API 連携、最終的に e-Tax XML 形式での書き出しまで進めば、会計事務所さんの業務がほぼ完結します。

私の稼働は月 20〜30 時間想定、平日 12 時から 16 時で柔軟に対応可能で、Claude Code を使った高速プロトタイピングが得意です。

もし方向性として合いそうでしたら、技術質問や実データを見せていただいて、詳細スコープのすり合わせをさせていただきたいです」

## 5. クロージング（質疑への切り替え）

「ご質問ありましたらお願いします」

---

## トーン注意点

- Shun の note.com 記事のような「直接的・断定的」なトーンは封印し、会計事務所案件らしく**誠実・冷静**を最優先
- 「すごい」「画期的」「最先端」など盛る表現は使わない
- 「業務に役立つかどうか」を一貫して中心に置く
- カバンから帳票出てきた、くらいの実物感で見せる
